New Features

  • Re-enable the missing-package banner: A new Code menu and command-palette entry re-enables the editor's missing-package banner for a file after it has been dismissed, and the banner's dismissal label is now "Don't show for this file" to make the per-file scope explicit.
  • Unicode section delimiters: Em dashes and box-drawing characters are now supported as native R code section delimiters.
  • Mouse-wheel tab switching preference: A new "Change active editor tab with mouse wheel" preference (General > Basic > Other) disables switching the active editor tab by scrolling the mouse wheel over the tab bar.
  • Project-specific editor themes: A new Appearance pane in Project Options sets a project-specific editor theme; leaving it at "(Default)" uses the global theme. A new "Ignore project-specific appearance settings" option (Global Options > Appearance) keeps the global editor theme even when a project sets its own.
  • Posit Assistant — incremental code execution: Posit Assistant now runs R code expression by expression, interleaving each expression with its output in the chat pane, rather than running the whole block at once and showing all output at the end.
  • Native pipe by default: The Insert Pipe Operator command (Ctrl+Shift+M / Cmd+Shift+M) now inserts the native R pipe operator |> by default; the magrittr pipe %>% can be restored via the "Use R's native pipe operator, |>" preference.
  • Find in Files toolbar refinements: The Find in Files refresh button has moved to the right side of the toolbar, matching the convention used by other panes, and the replace preview's match highlight colors are now theme-aware so they no longer look out of place under dark editor themes.
  • Stricter project trust: A new project-trust-required session option (off by default) treats all projects as untrusted by default — including projects in the user's home directory — prompting users to trust each project when opened even if it contains no auto-executing files. The visual editor is now disabled in untrusted projects, whether or not this option is set; only the source editor is available for markdown documents.
  • Air formatting without air.toml: When "Use Air for code formatting" is enabled, Air now formats R documents even when no air.toml file is present, using the editor's indentation settings (indent style and width). The previous behavior of formatting only in projects containing an air.toml can be restored with the new "Only use Air when an air.toml file is found" option (Global Options > Code > Formatting).
